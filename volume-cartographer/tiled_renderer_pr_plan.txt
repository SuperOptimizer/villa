Goal:   Break into ~26 small PRs that never leave main broken.

The changes fall into three layers:
  1. Core library (~30 new files) — cache, VcDataset, types, utilities
  2. CLI apps (~15 files) — migrate from z5/ChunkCache to VcDataset/TieredChunkCache

Key constraint: new APIs are added alongside old ones first, then old ones removed.

Decision: The SuperOptimizer/utils repo is NOT vendored via FetchContent. Instead,
the specific files needed (zarr.hpp, tensor.hpp, json.hpp + .cpp files) will be
copied directly into the VC source tree when needed (PR 6).
